# TackleBox v0.1.0

## Release Identity

- release tag: `v0.1.0`
- release date: `2026-05-23`
- target product: `products/skr2-f429`
- release type: `reference`

## Summary

`v0.1.0` is the first publishable TackleBox reference release for the BigTreeTech SKR 2 STM32F429 platform. It establishes the named stack boundaries (`BootAnchors`, `KeelWare`, `Boatswain`, `Semaphorio`, and `SpyGlass`), provides a validated USB CDC recovery and runtime command path, packages deterministic release artifacts, and ships the first flashless Boatswain update contract as a configuration-profile installer.

## Known Limitations

- TMC verification on the current reference bench still reports `verify=0` / `ifcnt_valid=0` even after aligning the firmware read path with the shared `0x05` sync-byte protocol and re-testing live, so driver-backed motion remains a constrained path and should be treated as bring-up status, not a finished motion subsystem.
- Bench motion is substantially closer after live calibration at `AXIS.TRAVEL_UM_PER_ROTATION=15900`; on the current reference setup, `MOVEABS -20126` landed within about `2 mm` of a `100 mm` marker while travel away from home remained in the negative direction, but that mechanics result does not remove the TMC bring-up limitation.
- Independent homing and move feedrates are now live. On the current reference bench, `MOTION.HOME_FEEDRATE_MM_PER_MIN=600` and `MOTION.MOVE_FEEDRATE_MM_PER_MIN=1500` produced the best smooth and nearly silent motion observed so far, while `2400 mm/min` remained functional but noticeably rougher and likely needs later TMC chopping or quiet-step tuning.
- The self-hosted GitHub hardware workflow exists in `.github/workflows/hardware-validation.yml` but has not yet been executed on a real GitHub runner.
- Multi-board support is still architectural intent rather than a demonstrated second-board implementation.
- `Boatswain` v1 packages support runtime config profiles only, not executable on-device module updates.
- Some repository paths and symbols still carry the historical `keyswitch` naming during the compatibility transition.

## Upgrade Guidance

1. Back up any existing bootloader region before replacing a resident bootloader.
2. Flash the BootAnchor if the board does not already have the validated CDC bootloader installed.
3. Flash the application image with `products/skr2-f429/flash-app.ps1`.
4. Apply a Boatswain config package if the target bench profile requires it.
5. Run `products/skr2-f429/validate-bootanchor.ps1` and `products/skr2-f429/validate-runtime.ps1` on the board.

## Recommended Push Notes

When publishing `v0.1.0`, describe it as a reference-board release, not as a general multi-board production release. The current release is suitable for public consumption when its limitations are stated explicitly.
